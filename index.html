Five lines to your first database

js 
import { qeed } from 'qeed';

const db = qeed({ path: './data.db' });

db.set('user:1', { name: 'Alice', role: 'admin', score: 95 });
db.set('session:abc', { userId: 1 }, { ttl: 30_000 });

const rec = db.get('user:1');
console.log(rec.value.name);    // 'Alice'
console.log(rec.createdAt);     // Unix ms

const admins = db.query()
  .where('role', 'eq', 'admin')
  .sort('score', 'desc')
  .limit(10)
  .get();

db.close();

When to use Qeed

✓ Good fit
  • Single-process Node.js services
  • Persistent caching layer
  • Session and config storage
  • Local-first applications
  • Edge / embedded deployments
  • Dev tooling and CLI apps
✗ Not designed for
  • Multi-process write concurrency
  • Horizontal scaling across machines
  • Encryption at rest (use SQLCipher)
  • Storing cryptographic secrets
  • Replacing a production RDBMS
