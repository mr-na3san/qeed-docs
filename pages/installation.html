Installation

Qeed requires Node.js 18 or later and compiles one native addon (better-sqlite3) on first install. No other runtime dependencies exist.

npm

bash
npm install qeed

yarn / pnpm / bun

bash
yarn add qeed
pnpm add qeed
bun add qeed

Build requirements

better-sqlite3 is a native C++ addon. The build toolchain must be present:

PlatformRequirement
Linux / WSLbuild-essential, python3
macOSXcode Command Line Tools: xcode-select --install
WindowsVisual Studio Build Tools with "Desktop development with C++"
💡
Prebuilds available. better-sqlite3 ships precompiled binaries for common Node.js LTS versions on Linux, macOS, and Windows. If a prebuild matches your platform, no compiler is needed.

ESM import

js
import { qeed, Qeed } from 'qeed';
import { ValidationError, IOError } from 'qeed';

CommonJS require

js
const { qeed, Qeed } = require('qeed');
const { ValidationError, IOError } = require('qeed');
✅
No dual-package hazard. Qeed ships a single CommonJS source with an ESM shim via createRequire. Both import styles load the same code — one module instance regardless of how it is imported.

TypeScript

Qeed ships JSDoc typings. TypeScript users get full IntelliSense without a separate @types package. Add "checkJs": true to your tsconfig.json to enable type checking of JavaScript that imports Qeed.

Verify your install

js
import { qeed } from 'qeed';

const db = qeed({ path: ':memory:' });
db.set('hello', 'world');
console.log(db.get('hello').value); // 'world'
db.close();
