Quick Start

You'll have a persistent, queryable key-value store running in under two minutes.

Open a database

The qeed() factory opens (or creates) a SQLite file, applies WAL mode, initialises the schema, and returns a ready-to-use database instance.

js
import { qeed } from 'qeed';

// File database — persists across restarts
const db = qeed({ path: './myapp.db' });

// In-memory — ideal for tests, discarded on close
const mem = qeed({ path: ':memory:' });
💡
Always call db.close() when your process exits to flush the WAL and release the file lock. Use process.on('exit', () => db.close()) as a safety net.

Basic CRUD

js
// Store any JSON-serializable value
db.set('user:1', { name: 'Alice', role: 'admin', score: 95 });
db.set('config', { theme: 'dark', lang: 'en' });
db.set('count', 42);
db.set('flag', true);
db.set('empty', null);

// Read a record — returns null if missing or expired
const rec = db.get('user:1');
rec.value;      // { name: 'Alice', role: 'admin', score: 95 }
rec.key;        // 'user:1'
rec.namespace;  // 'default'
rec.collection; // 'default'
rec.createdAt;  // Unix ms
rec.updatedAt;  // Unix ms
rec.expiresAt;  // null (no TTL set)
rec.meta;       // null

// Check existence
db.has('user:1');        // true
db.has('nonexistent');   // false

// Update — set() is always an upsert
db.set('user:1', { name: 'Alice', role: 'superadmin', score: 99 });

// Delete — returns true if the record existed
db.delete('config');     // true
db.delete('config');     // false (already gone)

The record object

Every get() returns a frozen record object. Attempting to mutate it throws in strict mode.

FieldTypeDescription
keystringThe stored key
valueanyDeserialized JSON value
namespacestringNamespace the record lives in
collectionstringCollection within the namespace
createdAtnumberUnix ms — set on first write, never updated
updatedAtnumberUnix ms — updated on every write
expiresAtnumber | nullUnix ms expiry, or null for no TTL
metaobject | nullOptional metadata blob

TTL and expiration

TTL is specified in milliseconds. Expired keys return null from get(), false from has(), and are excluded from all queries — without any per-key timer overhead.

js
// Expires in 30 seconds
db.set('session:abc', { userId: 1 }, { ttl: 30_000 });

// Extend TTL on an existing key
db.ttl('session:abc', 60_000);    // now expires in 60s from now

// Remove TTL — key lives forever
db.persist('session:abc');

// Expired keys vanish automatically
await new Promise(r => setTimeout(r, 31_000));
db.get('session:abc');  // null — expired
⚠️
TTL precision. Expiry is checked at read time. Expired records are physically deleted by a background sweep (default every 60 seconds). They are never returned, but they do occupy disk space until swept.

Metadata

Store arbitrary metadata alongside a value — useful for audit trails, versioning, or source tracking without polluting the value itself.

js
db.set('invoice:7', { total: 249.00, currency: 'USD' }, {
  meta: { source: 'stripe', version: 3, createdBy: 'api' }
});

const rec = db.get('invoice:7');
rec.value;  // { total: 249.00, currency: 'USD' }
rec.meta;   // { source: 'stripe', version: 3, createdBy: 'api' }

Namespaces and collections

Namespaces and collections provide isolation at the storage level — the same key can exist independently in each. Scoped instances share the database connection, cache, events, and metrics.

js
const db = qeed({ path: './app.db', namespace: 'myapp' });

// Scoped to myapp/users
const users = db.collection('users');
users.set('u:1', { name: 'Alice' });

// Scoped to myapp/posts
const posts = db.collection('posts');
posts.set('u:1', { title: 'Hello' });   // same key, different collection

users.get('u:1').value;  // { name: 'Alice' }
posts.get('u:1').value;  // { title: 'Hello' }

// Cross-namespace isolation
const tenantA = db.namespace('tenant_a').collection('users');
const tenantB = db.namespace('tenant_b').collection('users');
tenantA.set('u:1', { plan: 'pro' });
tenantB.set('u:1', { plan: 'free' });
tenantA.get('u:1').value;   // { plan: 'pro' }
tenantB.get('u:1').value;   // { plan: 'free' }

// Bulk operations
users.flushCollection();             // delete all users records
db.namespace('tenant_a').flushNamespace();  // delete all of tenant_a
users.deletePrefix('u:');            // delete keys starting with 'u:'

Dot-notation field access

Read a single nested field without deserializing the whole value:

js
db.set('profile:1', {
  user: { name: 'Bob', address: { city: 'NYC', zip: '10001' } }
});

const rec = db.get('profile:1', { field: 'user.address.city' });
rec.value;  // 'NYC'

Batch operations

Run multiple operations in a single atomic transaction — either all succeed or all roll back.

js
db.batch([
  { type: 'set', key: 'product:1', value: { name: 'Widget', stock: 100 } },
  { type: 'set', key: 'product:2', value: { name: 'Gadget', stock: 50 }, options: { ttl: 86_400_000 } },
  { type: 'del', key: 'product:discontinued' },
]);
// All three operations applied, or none.
